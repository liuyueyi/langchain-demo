# 文章写作流程

四 步完成高质量公众号文章：搜索资料 → 撰写文章 → 生成标题 → 排版优化

## Step 1: 搜索资料

使用 WebSearch 工具搜索主题相关资料：

```
搜索要求：
1. 并行搜索多个来源（官方文档、X/Twitter、Reddit、技术论坛）
2. 优先获取最新资料（当月/当季）
3. 可启动多个并行 Task 加速搜索
4. 深度总结搜索结果
```

## Step 2: 撰写文章

**必须先读取用户的 `./prompt.pt` 获取写作风格**

技术写作要求：
- 每个抽象概念都要有具体例子
- 复杂的逻辑用比喻或类比解释
- 提供可运行的代码片段
- 说明适用场景和限制条件
- 准备 2-3 个备选标题

## Step 3: 生成标题

生成 5 个爆款标题，特点：
- **效率对比型**：直击读者痛处（"效率暴涨10倍"）
- **深度解析型**：深入理解{技术}：{核心特点}的设计哲学
- **问题解决型**：解决{具体问题}的{数字}种技术方案对比
- **实战指南型**：{技术栈}实战：从零搭建{项目类型}
- **原理剖析型**：{技术名词}的工作原理：{有趣的角度}

## Step 4: 排版优化

优化建议：
1. **段落结构**：每段 3-5 行，重要数据单独成段加粗
2. **配图位置**：标题下方封面图、步骤后效果图、结尾总结图
3. **代码块**：前后留白，语法高亮
4. **金句**：单独成段，增强记忆点

金句模板：
- "好的架构不是设计出来的，而是演化出来的。"
- "代码写给人看，顺便给机器执行。"
- "没有银弹，只有合适场景下的合适技术。"
- "过早优化是万恶之源，但完全不优化是另一种恶。"
- "理解原理比记住API更重要。"
- "可运行的烂代码，好过完美的不可运行代码。"

## 示例

用户请求：
```
写一篇关于"LangChain应用教程"：
```

执行流程：
1. 搜索 LangChain 相关资料
2. 读取 prompt.pt 获取写作风格
3. 撰写图文并茂的教程文章
4. 生成 5 个爆款标题供选择
5. 提供排版和配图建议
